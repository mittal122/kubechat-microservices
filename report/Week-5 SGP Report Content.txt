Tasks Completed

Task 3:

Task 4:
Upgraded message status model from boolean seen state to enum lifecycle: sent, delivered, seen.

Task 5:
Implemented instant delivered state when receiver is online at send time.

Task 6:
Implemented auto-delivery update for previously sent messages when receiver reconnects.

Task 7:
Enabled preview mode behavior so opening overlay does not automatically mark messages as seen.

Challenges Faced
Challenge 1:
Keeping message status synchronized across many socket connections for the same user.

Resolution:
Introduced user-to-socket array mapping and broadcast updates to all active sessions.

Challenge 2:
Avoiding incorrect seen updates during chat preview interactions.

Resolution:
Separated preview and focus behavior so seen updates trigger only on explicit chat focus action.

Learnings and Skills Acquired
